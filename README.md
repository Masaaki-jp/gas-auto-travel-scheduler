# Survival DX - 自動経路登録スケジューラ

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Googleカレンダーに登録された予定を読み取り、最適な移動経路と所要時間を自動計算してカレンダーにブロックを自動生成するGoogle Apps Script（GAS）です。

人間の意思決定や行動管理をシステムに委ねる**「自律型データエージェント」**構想の基盤として開発しています。高価なサーバーは不要。クラウド上のリソースと公開APIを「術」として組み合わせ、限られたリソースで生存能力・生産性を最大化する「Survival DX」の理念を体現するプロジェクトです。

---

## 🚀 最新アップデート：Phase 2 (V2.0) 完全自律型ルーティング

V2.0では、気象API（OpenWeather / Yahoo!気象情報）と連携した環境データレイヤーを導入し、**「気象ハザードに応じた交通手段の動的変更」**を実現しました。予定の時間を計算するだけでなく、「いかに安全に目的地へ到達するか」をシステムが自律的に判断します。

* 🚨 **命の危険（猛暑・酷暑）:** 徒歩・自転車を排除し、強制的に車（または公共交通機関）を選択。
* 💨 **強風リスク回避:** 自転車ルートをスキップし、安全な代替手段へフォールバック。
* ☔ **雨天時の最適化:** 徒歩区間が15分以上の場合は、電車ルートから車へ自動スイッチ。

### 🎨 カレンダー上の視覚的メトリクス（カラーコード）
システムが選択した交通手段は、カレンダー上で色分けされ、視覚的に瞬時に把握できます。
* 🟨 **[Walk] (黄色)**: 徒歩（15分未満）
* 🟩 **[Bicycle] (緑色)**: 自転車（15分以上30分未満 / 強風時は回避）
* 🟦 **[Train/Bus] (青色)**: 公共交通機関（雨天時や長距離移動）
* 🟥 **[Car] (赤色)**: 自動車（猛暑・酷暑時、または駅遠物件への雨天移動）

## 🌱 Phase 1 (V1.0) 基本の自動ルーティング機能

V1.0から実装されている、GASとGoogle Maps API（標準搭載）のみで稼働する強力な基礎エンジンです。

* **未来予定の検知:** 1ヶ月先までの「場所」ラベル付き予定を自動で監視。
* **動的ルーティング:** 一日の始まりは必ず「拠点」とし、連続する予定がある場合は「前の予定の場所」から次の経路を計算。
* **帰還の自動化:** その日の任務（予定）がすべて終わったら、最終地点から拠点への「帰宅スケジュール」を自動生成。
* **重複登録の防止:** 15分おき等の定期実行（トリガー）を行っても、同じ移動予定を重複して作らない安全設計。

## ⚙️ 導入手順

### Step 1: 共通の基本設定
1. Google Apps Script ([script.google.com](https://script.google.com/)) を開き、新しいプロジェクトを作成します。
2. 使用するフェーズのコード（`v1` または `v2`）をコピーしてエディタに貼り付けます。
3. コード内の `HOME_ADDRESS = '...';` の部分を、ご自身の起点となる住所（Googleマップで検索可能な形式）に変更してください。
4. 時計マークの「トリガー」メニューから、メイン関数（例：`automateHazardAwareTravelSchedule`）を「時間主導型（例：毎日早朝、または15分おき）」で設定します。
5. 初回実行時にGoogleの認証画面が出ますので、承認を行ってください。

### Step 2: Phase 2 (V2.0) を利用する場合の追加設定
V2.0を稼働させるには、環境データを収集する気象カレンダーが別途必要です。
1. コード内の `WEATHER_CALENDAR_ID = '...';` を、ご自身の天気情報が登録されているカレンダーのIDに変更します。
2. ハザード検知のキーワード（`HAZARD_MAP`）を、お使いの天気カレンダーの予定タイトルに合わせて調整してください。
※ 気象データ（OpenWeather / Yahoo!気象情報）をカレンダーに自動登録するスクリプトは、本リポジトリとは別に稼働させておく必要があります。
